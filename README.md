# メディアクエリとコンテナクエリについて

## メディアクエリ (Media Queries)

**「ビューポート（画面）のサイズ」** に基づいてスタイルを変える仕組みです。

```css name=media-query-example.css
/* 画面幅が 768px 以下のとき */
@media (max-width: 768px) {
  .card {
    font-size: 14px;
    flex-direction: column;
  }
}
```

### 特徴
- ✅ ブラウザのウィンドウサイズを基準にする
- ✅ 長年使われてきた実績がある
- ❌ **コンポーネント自身のサイズ**は考慮できない
- ❌ 同じコンポーネントを複数箇所で使い回すのが難しい

---

## コンテナクエリ (Container Queries)

**「親要素（コンテナ）のサイズ」** に基づいてスタイルを変える仕組みです。CSS の比較的新しい機能です。

```css name=container-query-example.css
/* 親要素をコンテナとして定義 */
.wrapper {
  container-type: inline-size;
  container-name: card-container;
}

/* コンテナの幅が 500px 以上のとき */
@container card-container (min-width: 500px) {
  .card {
    font-size: 18px;
    flex-direction: row;
  }
}
```

### 特徴
- ✅ **コンポーネント単位**でレスポンシブにできる
- ✅ 同じコンポーネントをサイドバーにもメインエリアにも使い回せる
- ✅ 現代ブラウザはほぼ全てサポート済み（2023年〜）
- ❌ 古いブラウザは非対応

## 🆚 どちらを使うべき？

| 観点 | メディアクエリ | コンテナクエリ |
|---|---|---|
| 基準 | 画面（ビューポート）のサイズ | 親要素のサイズ |
| 向いている場面 | ページ全体のレイアウト切り替え | 再利用可能なコンポーネント |
| ブラウザ対応 | 全ブラウザ | モダンブラウザ（Chrome 105+, Firefox 110+, Safari 16+） |
| 柔軟性 | △ | ◎ |

## 現代のベストプラクティス

> **ページレイアウト → メディアクエリ**  
> **コンポーネント → コンテナクエリ**  
> の組み合わせが最もよく使われます。
